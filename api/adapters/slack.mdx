The Slack adapter implements the `PlatformAdapter` interface to enable Switchboard bots to connect to Slack using the Bolt framework.

## Installation

```bash
npm install @aarekaz/switchboard-slack
```

## SlackAdapter

The main adapter class that handles Slack platform integration with built-in message context caching.

### Constructor

```typescript

```

<ParamField path="config" type="SlackConfig" optional>
  Optional configuration for the Slack adapter
  
  <Expandable title="properties">

      Cache TTL in milliseconds (default: 1 hour)

<Tabs>

  <Tab title="Events API (Production)">
    Recommended for production. Requires `botToken` and `signingSecret`.
    

  </Tab>
</Tabs>

#### sendMessage

Send a message to a Slack channel.

```typescript
async sendMessage(
  channelId: string,
  text: string,
  options?: SendMessageOptions
): Promise<Result<UnifiedMessage>>
```

<ParamField path="channelId" type="string" required>
  Slack channel ID where the message will be sent
</ParamField>

<ParamField path="text" type="string" required>
  Message text content
</ParamField>

<ParamField path="options" type="SendMessageOptions" optional>
  Platform-specific message options
  
  <Expandable title="properties">
    <ParamField path="threadId" type="string" optional>
      Thread timestamp to reply in thread
    </ParamField>
    
    <ParamField path="slack.blocks" type="unknown[]" optional>
      Slack Block Kit blocks for rich formatting
    </ParamField>
    
    <ParamField path="slack.unfurl_links" type="boolean" optional>
      Whether to unfurl links in the message
    </ParamField>
    
    <ParamField path="slack.unfurl_media" type="boolean" optional>
      Whether to unfurl media in the message
    </ParamField>
    
    <ParamField path="slack.metadata" type="unknown" optional>
      Message metadata
    </ParamField>
    
    <ParamField path="slack.thread_ts" type="string" optional>
      Thread timestamp (alternative to `threadId`)
    </ParamField>
  </Expandable>
</ParamField>

**Returns:** `Result<UnifiedMessage>` - Success result with the sent message, or error

**Example:**

```typescript
const result = await adapter.sendMessage(
  'C1234567890',
  'Hello Slack!',
  {
    slack: {
      blocks: [
        {
          type: 'section',
          text: {
            type: 'mrkdwn',
            text: '*Hello* from Switchboard!'
          }
        }
      ]
    }
  }
);
```

#### editMessage

Edit an existing message.

```typescript
async editMessage(
  messageRef: MessageRef,
  newText: string
): Promise<Result<UnifiedMessage>>
```

<ParamField path="messageRef" type="MessageRef" required>
  Message reference (full message object or message ID)
</ParamField>

<ParamField path="newText" type="string" required>
  New message text
</ParamField>

**Returns:** `Result<UnifiedMessage>` - Success result with the edited message, or error

<Warning>
  On Slack, editing messages by ID only works if the message is in the cache. Pass the full message object for reliability:
  
  - `bot.editMessage(message, "text")` ✅ Works reliably
  - `bot.editMessage(message.id, "text")` ❌ May fail if cache expired
  
  Messages are cached for 1 hour by default (configurable via `cacheTTL`).
</Warning>

#### onEvent

Register an event handler for Slack events.

```typescript
onEvent(handler: (event: UnifiedEvent) => void | Promise<void>): void
```

<ParamField path="handler" type="(event: UnifiedEvent) => void" required>
  Event handler function that receives unified events
</ParamField>

**Supported events:**
- `message` - New message received
- `reaction` - Reaction added or removed

**Example:**

```typescript
adapter.onEvent((event) => {
  if (event.type === 'message') {
    console.log('Message received:', event.message.text);
  } else if (event.type === 'reaction') {
    console.log('Reaction:', event.action, event.emoji);
  }
});
```

## Message caching

The Slack adapter uses an LRU cache to store message context (channel ID, thread ID) because Slack's API requires the channel ID for most operations, but only provides the timestamp in some contexts.

### Cache configuration

```typescript
const adapter = new SlackAdapter({
  cacheSize: 1000,     // Store up to 1000 messages
  cacheTTL: 3600000    // Cache for 1 hour
});
```

### Cache behavior

- Messages are automatically cached when sent or received
- Cache is cleared on disconnect
- Cache statistics are logged every 1000 operations in development

### Best practices

<Tip>
  Always pass the full message object instead of just the ID to avoid cache misses:
  
  ```typescript
  // ✅ Recommended
  await bot.editMessage(message, 'Updated text');
  await bot.addReaction(message, '👍');
  
  // ❌ May fail if cache expired
  await bot.editMessage(message.id, 'Updated text');
  await bot.addReaction(message.id, '👍');
  ```
</Tip>

## Usage example

```typescript
import { createBot } from '@aarekaz/switchboard';
import '@aarekaz/switchboard-slack';

const bot = createBot({
  platform: 'slack',
  credentials: {
    botToken: process.env.SLACK_BOT_TOKEN,
    appToken: process.env.SLACK_APP_TOKEN  // Socket Mode
  },
  config: {
    cacheSize: 2000,
    cacheTTL: 7200000  // 2 hours
  }
});

bot.onMessage(async (message) => {
  if (message.text === 'ping') {
    const result = await bot.sendMessage(
      message.channelId,
      'pong!',
      {
        slack: {
          blocks: [
            {
              type: 'section',
              text: {
                type: 'mrkdwn',
                text: '*Pong!* :ping_pong:'
              }
            }
          ]
        }
      }
    );
  }
});

await bot.start();
```
